# Claude Code Task: Phase 2 — Supervised Fine-Tuning on Trading Q&A

## Context

Phase 1 (continued pretraining) is complete. A LoRA adapter at
`./phi3-trading-lora/adapter/` has been trained on the raw trading corpus,
teaching Phi-3 Mini the domain language.

Phase 2 applies supervised fine-tuning (SFT) on structured Q&A pairs so the
model learns to *respond* to trading questions in the right analytical format.

This folder contains:

- `train.py` — autoresearch reference (read-only, do NOT modify)
- `finetune_phi3.py` — Phase 1 CPT script (do NOT modify)
- `build_dataset.py` — Q&A dataset builder (read-only, do NOT modify)
- `trading_qa.jsonl` — generated by build_dataset.py (if it exists already)

**Create a new file: `finetune_phi3_sft.py`** — do NOT overwrite finetune_phi3.py.

## What this script does

1. Loads Phi-3 Mini base model
2. Merges the Phase 1 CPT LoRA adapter into the base weights
3. Applies a fresh LoRA adapter for SFT training
4. Trains on `trading_qa.jsonl` (Phi-3 ChatML format Q&A pairs)
5. Saves the SFT LoRA adapter
6. Runs a smoke test with trading prompts
7. Exports merged model to GGUF Q4_K_M for Raspberry Pi deployment

## Hardware

- GPU: NVIDIA RTX 5060 Ti 16 GB VRAM (Blackwell, CC 12.0)
- Full bf16 LoRA (not QLoRA)
- OS: Windows, PowerShell

## Step-by-step implementation

### 1. Model loading — merge Phase 1 then apply fresh LoRA

```python
from unsloth import FastLanguageModel
from pathlib import Path

MODEL_NAME      = "unsloth/Phi-3-mini-4k-instruct"
CPT_ADAPTER_DIR = Path("./phi3-trading-lora/adapter")
MAX_SEQ_LEN     = 2048

def load_model_with_cpt_adapter(use_qlora: bool = False):
    """
    Load Phi-3, merge the Phase 1 CPT adapter, then apply a fresh LoRA for SFT.
    """
    dtype = None if use_qlora else torch.bfloat16
    load_in_4bit = use_qlora

    # Step 1: Load base model WITH the CPT adapter
    print(f"Loading base model + CPT adapter from {CPT_ADAPTER_DIR}")
    model, tokenizer = FastLanguageModel.from_pretrained(
        model_name     = str(CPT_ADAPTER_DIR),  # Unsloth loads adapter on top of base
        max_seq_length = MAX_SEQ_LEN,
        dtype          = dtype,
        load_in_4bit   = load_in_4bit,
    )

    # Step 2: Merge CPT adapter into base weights permanently
    # This folds Phase 1 knowledge into the base so Phase 2 LoRA trains on top
    print("Merging CPT adapter into base weights ...")
    model = model.merge_and_unload()

    # Step 3: Apply fresh LoRA for SFT
    # SFT LoRA is SMALLER rank than CPT — we're learning response format,
    # not domain knowledge (that's already baked in from Phase 1)
    model = FastLanguageModel.get_peft_model(
        model,
        r                          = 8,       # lower rank — SFT is lighter
        target_modules             = ["q_proj", "k_proj", "v_proj", "o_proj",
                                      "gate_proj", "up_proj", "down_proj"],
        lora_alpha                 = 16,      # alpha = 2x rank
        lora_dropout               = 0.05,
        bias                       = "none",
        use_gradient_checkpointing = "unsloth",
        random_state               = 42,
        use_rslora                 = True,
    )

    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    total     = sum(p.numel() for p in model.parameters())
    print(f"SFT trainable params: {trainable:,} ({100*trainable/total:.2f}%)")

    return model, tokenizer
```

### 2. SFT hyperparameters — adjusted from autoresearch

SFT needs a gentler config than CPT. The domain knowledge is already in the
weights — we're only teaching response format now.

```python
# SFT-specific — lower LR, fewer epochs, small rank
SFT_LEARNING_RATE   = 1e-4       # 5x lower than CPT — format learning is easier
SFT_WARMUP_RATIO    = 0.03       # small warmup — SFT on merged weights benefits from it
SFT_WEIGHT_DECAY    = 0.01       # lighter regularisation
SFT_EPOCHS          = 3          # Q&A datasets are small, 3 passes is enough
SFT_BATCH_SIZE      = 2
SFT_GRAD_ACCUM      = 4          # effective batch = 8

# These carry over from autoresearch unchanged
ADAM_BETAS           = (0.8, 0.95)
ADAM_EPSILON         = 1e-10
FINAL_LR_FRAC        = 0.1       # cosine floor at 10%
```

### 3. Data loading — use build_dataset.py output

```python
from datasets import load_dataset
from unsloth.chat_templates import get_chat_template

def load_sft_dataset(dataset_path: Path, tokenizer, val_fraction: float = 0.1):
    """
    Load trading_qa.jsonl in Phi-3 ChatML format.
    Each row has a 'messages' field with system/user/assistant turns.
    """
    raw = load_dataset("json", data_files=str(dataset_path), split="train")

    # Apply Phi-3 chat template
    tokenizer = get_chat_template(tokenizer, chat_template="phi-3")

    def format_example(example):
        text = tokenizer.apply_chat_template(
            example["messages"],
            tokenize=False,
            add_generation_prompt=False,
        )
        return {"text": text}

    formatted = raw.map(format_example, remove_columns=raw.column_names)
    split = formatted.train_test_split(test_size=val_fraction, seed=42)

    print(f"SFT train examples: {len(split['train'])}")
    print(f"SFT val examples:   {len(split['test'])}")
    return split["train"], split["test"]
```

### 5. Cosine-with-floor scheduler callback

Same as Phase 1 — reuse the callback:

```python
import math
from transformers import TrainerCallback

class CosineFloorScheduler(TrainerCallback):
    def on_step_begin(self, args, state, control, optimizer=None, **kwargs):
        if optimizer is None or state.max_steps == 0:
            return
        progress = state.global_step / state.max_steps
        cosine = 0.5 * (1.0 + math.cos(math.pi * progress))
        multiplier = FINAL_LR_FRAC + (1.0 - FINAL_LR_FRAC) * cosine
        for group in optimizer.param_groups:
            if "initial_lr" not in group:
                group["initial_lr"] = group["lr"]
            group["lr"] = group["initial_lr"] * multiplier
```
